# 市场交易时段与开盘/休市判断逻辑（北京时间）

> 本文档统一整理 A股（指数/股票/ETF）、港股、美股的品种识别方法与交易时段判断逻辑。
> **所有时间均以北京时间（UTC+8）为准**；美股因夏令时/冬令时换算不同，单独列出两套时间。
> 文末附本项目现有实现的代码索引，以及现状与本文档完整规则之间的差距。

---

## 1. 市场与品种识别

### 1.1 市场类型（项目内枚举 `MARKET_TYPE`）

| 市场 | 枚举值 | 说明 |
|---|---|---|
| A股 | `CN` | 沪、深交易所（北交所暂未支持） |
| 港股 | `HK` | 港交所，香港时间 = 北京时间，无时差 |
| 美股 | `US` | NYSE / NASDAQ，美东时间，需夏冬令时换算 |

### 1.2 品种类型（项目内枚举 `STOCK_TYPE`）

`stock`（个股）、`index`（指数）、`etf`（ETF）。

搜索时由腾讯搜索接口返回的类型字段判断（`searchStock`，`src/api/stock.js:541`）：

- `GP.I` 开头 → 指数
- 等于 `ETF` → ETF
- 其余 `GP` 开头 → 个股
- 其它（窝轮、牛熊证等衍生品）→ 过滤掉

### 1.3 A股代码规则（6位数字）

项目内简化规则（`getCNExchangePrefix`，`src/api/stock.js:217`）：代码以 `5 / 6 / 9 / 11 / 13` 开头 → 沪市 `sh`，否则 → 深市 `sz`。

完整对照（供扩展参考）：

| 代码段 | 市场 | 品种 |
|---|---|---|
| 600 / 601 / 603 / 605 | 沪 | 主板个股 |
| 688 / 689 | 沪 | 科创板个股 / CDR |
| 000 / 001 / 002 / 003 | 深 | 主板个股 |
| 300 / 301 | 深 | 创业板个股 |
| 51x / 56x / 58x（5开头） | 沪 | ETF / 基金 |
| 159 | 深 | ETF |
| 110 / 111 / 113 | 沪 | 可转债 |
| 123 / 127 / 128 | 深 | 可转债 |
| 900 / 200 | 沪 / 深 | B股 |
| 000xxx（带 `sh` 前缀） / 399xxx | 沪 / 深 | 指数（如 sh000001 上证指数、sz399001 深成指、sz399006 创业板指） |
| 8xxxxx / 4xxxxx | 北交所 | 个股（**现有规则会误判为 sz，暂不支持**） |

⚠️ 注意：指数 `000001`（上证指数）与个股 `000001`（平安银行）代码相同，**必须靠 `type` + 交易所前缀区分**，不能只看代码。

### 1.4 港股 / 美股代码规则

- **港股**：5位数字（如 `01810` 小米、`00700` 腾讯）；指数为字母代码（`HSI` 恒指、`HSCEI` 国企、`HSTECH` 恒生科技）。新浪行情前缀：个股 `rt_hk{code}`，指数 `rt_hk{INDEX}`（见 `getHKIndexQuote` 的映射表）。
- **美股**：1~5位字母（如 `AAPL`、`GOOGL`），可带类别后缀（`BRK.B`）；项目内请求前会去掉 `.` 后缀并小写，新浪前缀 `gb_{symbol}`。

## 2. 各市场交易时段总表（北京时间）

### 2.1 A股（周一至周五，法定节假日休市，无半日市）

| 时段 | 北京时间 | 说明 |
|---|---|---|
| 开盘集合竞价 | 09:15 – 09:25 | 09:15–09:20 可撤单，09:20–09:25 不可撤单，09:25 产生开盘价 |
| 静默期 | 09:25 – 09:30 | 可挂单不撮合 |
| 早盘连续竞价 | **09:30 – 11:30** | 盘中 |
| **午间休市** | **11:30 – 13:00** | 当日仍是交易日 |
| 午盘连续竞价 | **13:00 – 14:57** | 盘中 |
| 收盘集合竞价 | 14:57 – 15:00 | 沪深均有，15:00 产生收盘价 |
| 盘后固定价格交易 | 15:05 – 15:30 | 仅科创板/创业板，按收盘价成交 |

- A股**指数**与**ETF**的交易/发布时段与个股完全一致（09:30–11:30、13:00–15:00），用同一套判断即可。
- 项目内判断口径：盘中 = `09:30–11:30 ∪ 13:00–15:00`（含边界），不单独识别集合竞价。

### 2.2 港股（周一至周五，香港公众假期休市；香港时间 = 北京时间）

| 时段 | 北京时间 | 说明 |
|---|---|---|
| 开市前竞价 | 09:00 – 09:30 | 09:20 后不可撤单，随机对盘开市 |
| 早市 | **09:30 – 12:00** | 盘中 |
| **午间休市** | **12:00 – 13:00** | 当日仍是交易日 |
| 午市 | **13:00 – 16:00** | 盘中 |
| 收市竞价 | 16:00 – 16:10 | 16:08–16:10 随机收市，产生收盘价 |

港股特有情况：

- **半日市**：圣诞前夕（12-24）、新年前夕（12-31）、农历除夕，只有早市，**12:00 收市**（收市竞价 12:00–12:10），当天下午属于"当日已收盘"而非午间休市。
- **休市日与 A股不同**：香港公众假期（如复活节、佛诞、圣诞节翌日）A股照常交易；反之 A股的部分假期（如国庆后几天）港股照常。两市各自维护，不可混用。
- 恶劣天气（台风/黑雨）：自 2024-09-23 起港股**照常交易**，不再因天气休市。
- 港股**指数**（HSI/HSTECH 等）发布时段与个股盘中一致，复用同一判断。

### 2.3 美股（美东时间交易，按北京时间需区分夏令时/冬令时）

夏令时规则：**3月第二个周日 02:00（ET）开始，11月第一个周日 02:00（ET）结束**。

- 夏令时（EDT，UTC-4）：北京时间 = 美东时间 + 12 小时
- 冬令时（EST，UTC-5）：北京时间 = 美东时间 + 13 小时

| 时段 | 美东时间 | 北京时间（夏令时） | 北京时间（冬令时） |
|---|---|---|---|
| 夜盘（Overnight） | 20:00 – 次日 04:00 | 08:00 – 16:00 | 09:00 – 17:00 |
| 盘前（Pre-market） | 04:00 – 09:30 | 16:00 – 21:30 | 17:00 – 22:30 |
| **盘中（Regular）** | **09:30 – 16:00** | **21:30 – 次日 04:00** | **22:30 – 次日 05:00** |
| 盘后（After-hours） | 16:00 – 20:00 | 04:00 – 08:00 | 05:00 – 09:00 |

说明：

- 一个"美股交易日"在北京时间上**横跨两天**：例如美东周一的盘中，对应北京周一 21:30 至周二凌晨 04:00。**判断周末必须先换算成美东的星期几**，不能直接用北京的星期。
- 夜盘指 Blue Ocean 等 ATS 提供的隔夜交易（美东周日 20:00 起至周五 04:00），并非交易所官方时段，行情源覆盖有限，可按需展示。
- 盘前/盘后属于延长时段（Extended Hours），成交清淡、价格仅供参考；"当前价"应标注所属时段。
- **半日市**：感恩节次日、独立日前夕、平安夜等，13:00 ET 提前收盘（北京时间次日 01:00 夏 / 02:00 冬），盘后相应缩短。

## 3. 状态判断逻辑（状态机）

对任意一只标的，状态可归为以下几类（按北京时间 `now` 判断）：

```
trading        盘中（连续竞价中，最新价为实时价）
midday_break   午间休市（仅 A股/港股；当日仍是交易日）
pre_market     盘前（仅美股延长时段）
after_hours    盘后（仅美股延长时段）
overnight      夜盘（仅美股 ATS 时段）
closed         已收盘 / 当日休市（周末、节假日、收盘后）
```

### 3.1 通用判断流程（伪代码）

```
function getMarketSession(market, nowBeijing):
  # 第一步：换算到交易所本地时间
  #   CN/HK：本地时间 = 北京时间
  #   US   ：本地时间 = 北京时间 - 12h(夏令时) / -13h(冬令时)，星期也要随之换算
  local = toExchangeTime(market, nowBeijing)

  # 第二步：周末 → 当日休市
  if local.weekday in [周六, 周日]: return closed

  # 第三步：节假日 → 当日休市（无节假日日历时见 3.3 的兜底方案）
  if isHoliday(market, local.date): return closed

  # 第四步：按时段表匹配
  match market:
    CN: 09:30–11:30 / 13:00–15:00 → trading
        11:30–13:00               → midday_break
        其它                       → closed
    HK: 09:30–12:00 / 13:00–16:00 → trading（半日市日只认上午）
        12:00–13:00               → midday_break（半日市日为 closed）
        其它                       → closed
    US: 09:30–16:00 (ET)          → trading
        04:00–09:30 (ET)          → pre_market
        16:00–20:00 (ET)          → after_hours
        20:00–04:00 (ET)          → overnight
```

### 3.2 区分「午间休市」与「当日休市」

这是两个容易混淆但语义完全不同的状态：

| | 午间休市 | 当日休市 |
|---|---|---|
| 含义 | 今天是交易日，中午暂停 1~1.5 小时 | 今天整天不交易（周末/节假日/半日市下午） |
| 适用市场 | 仅 A股（11:30–13:00）、港股（12:00–13:00） | 所有市场 |
| 最新价含义 | 上午收盘的盘中价，下午还会变 | 上一交易日收盘价 |
| UI 处理（本项目） | "隐藏已闭市"时**仍然显示** | 隐藏 |

判断顺序必须是：**先排除周末/节假日，再看是否落在午休时间窗**。只看时间窗（如现有 `isCNMarketMiddayBreak`）会把"节假日的中午"误判为午间休市，需配合 3.3 的兜底。

### 3.3 节假日的兜底判断（本项目方案）

项目没有维护各市场节假日日历，采用**行情快照新鲜度**兜底（`resolveQuoteStatus`，`src/api/stock.js:83`）：

```
status = closed                      # 不在盘中时间窗 → 直接 closed
若在盘中时间窗：
  quoteDate = 行情 updateTime 中解析出的日期
  today     = 当前日期（美股用美东日期 getUSDateKey，A股/港股用北京日期）
  quoteDate == today → trading      # 今天有成交，是真交易日
  quoteDate != today → closed       # 时间窗内但行情还是昨天的 → 今天是节假日，当日休市
```

这同样能区分"节假日中午"与"交易日午休"：午休时上午刚有成交，`updateTime` 是今天；节假日 `updateTime` 停留在上一交易日。

- 美股日期解析兼容 `MM-DD-YYYY` 格式（新浪美股接口格式，`extractDateKey`）。
- 解析不出日期时默认放行（`trading`），宁可多显示不漏显示。

## 5. 现状与完整规则的差距（待扩展项）

1. **美股盘前/盘后/夜盘未实现**：`isUSMarketOpen` 只认 09:30–16:00 ET，延长时段一律 `closed`。若要展示盘前/盘后价，需按第 2.3 节时段表扩展出 `pre_market / after_hours / overnight` 状态，并接入对应行情字段。
2. **依赖本机时区**：所有判断用 `new Date()` 的本地时间，隐含"设备处于北京时间"的假设。设备时区非 UTC+8 时全部判错。更稳妥的做法是用 `Intl.DateTimeFormat` 按 `Asia/Shanghai` / `America/New_York` 取时分与星期。
3. **港股半日市未处理**：平安夜/除夕/新年前夕下午会被误判为午间休市后的"盘中"（实际无成交，靠行情新鲜度兜底部分缓解）。
4. **边界口径**：盘中判断含两端（如 11:30、15:00 整点算盘中），午休判断为开区间（11:31 起算午休），两者衔接处一致、无空洞，但 15:00:00–15:00:59 仍算盘中，与收盘竞价已结束的事实略有出入，影响可忽略。
5. **集合竞价时段未识别**：09:15–09:30（A股）、09:00–09:30（港股）按 `closed` 处理，如需展示竞价价格需单独加状态。
6. **北交所（8/4 开头）不支持**：会被 `getCNExchangePrefix` 误判为深市。
7. **节假日无日历**：完全依赖行情 `updateTime` 兜底，行情源格式变化会导致误判（解析失败时默认 `trading`）。
